Grille

La grille possède 12 colonnes et est basée sur Flexbox Grid.
Il y a 4 préfixes selon la taille de l'écran : sm, md, lg et xl. L'absence de préfixe permet d'appliquer un format pour toutes les tailles.

Pour que la grille fonctionne, l'élément parent doit avoir la classe .container.

Taille fixe

Pour forcer la taille des colonnes il suffit d'utiliser col-[préfixe]-[taille] (exemple : col-sm-6).
La taille doit être comprise entre 1 et 12, et une absence de préfixe permettra un affichage identique quel que soit le média utilisé.

Utilisez différents préfixes pour changer la disposition selon la taille de l'écran.

.col-md-8.col-3

Il est possible de déplacer une colonne vers la droite en utilisant le préfixe offset.

.col-offset-3.col-9

Titres

Toutes les tailles sont basées sur l'unité rem.

La police de base utilisée est Raleway hébergée par Google. Skeleton inclus les liens, la mise en gras, l'italique, et le soulignement.

Chaque titre possède une taille unique, ainsi que les propriétés letter-spacing, line-height, et margin.

Boutons

Les boutons viennent dans deux styles. L'élément <button> standard est simple, alors que le bouton .button-primary est coloré et mis en avant. Les styles de boutons sont appliqués sur les éléments de base, mais n'importe quel élément peut être transformé avec la classe .button.

Lien en forme de bouton
Lien en forme de bouton 
<!-- Boutons standards -->
<a class="button" href="#">Lien en forme de bouton</a>
<button>Elément button</button>
<input type="submit" value="input type submit">
<input type="button" value="input type button">

<!-- Boutons accentués -->
<a class="button button-primary" href="#">Lien en forme de bouton</a>
<button class="button-primary">Elément button</button>
<input class="button-primary" type="submit" value="input type submit">
<input class="button-primary" type="button" value="input type button">

Formulaires

Tous les champs, liste de sélection et boutons sont normalisés pour avoir une hauteur commune, afin de pouvoir les empiler ou les placer à côtés les uns des autres.

    <div class="row">
        <div class="col no-padding">
            <label for="exampleEmailInput">Votre adresse email</label>
            <input class="full-width" type="email" placeholder="test@mail.com" id="exampleEmailInput">
        </div>
        <div class="col no-padding" style="margin-left: 1.5rem;">
            <label for="exampleRecipientInput">Sujet de l'email</label>
            <select class="full-width" id="exampleRecipientInput">
                <option value="Option 1">Questions</option>
                <option value="Option 2">Remarques</option>
                <option value="Option 3">Demande de remboursement</option>
            </select>
        </div>
    </div>

    <label for="exampleMessage">Message</label>
    <textarea class="full-width" placeholder="Bonjour Skeleton, je souhaite ..."
        id="exampleMessage"></textarea>

        <input type="checkbox">
        <span class="label-body">M'envoyer une copie</span>
    </label>

    <input class="button-primary" type="submit" value="Envoyer">
</form>

Code

La stylisation de code est basique. Entourez votre code d'une balise <code> et il sera affiché comme ceci. Pour des blocs de code, entourez l'élément <code> avec un élément <pre>.

Pour utiliser une coloration syntaxique, la librairie HighlightJS fait des merveilles ! 

.une-classe {
    background-color: red;
}
<pre><code>
.une-classe {
  background-color: red;
}</code></pre>

@Media

Skeleton A11y utilise les directives CSS @media pour adapter l'affichage sur les différents écrans. Préférez configurer votre site pour mobile en premier (approche mobile first), puis inclure dans les directives @media les parties spécifiques aux plus grands terminaux.

  • Bureau HD: 1200px
  • Bureau: 992px
  • Tablette: 768px
  • Phablette: 576px
  • Mobile: 0px
/* CSS Mobile first */

/* Phablettes et + */
@media (min-width: 550px) {}

/* Tablettes et + */
@media (min-width: 750px) {}

/* Bureau et + */
@media (min-width: 1000px) {}

/* Bureau HD et + */
@media (min-width: 1200px) {}

Autres

Couleurs

Skeleton A11y utilise les variables globales CSS pour gérer les couleurs du thème. Les couleurs d'origine sont accessibles, mais il est aussi très simple de modifier le thème à sa convenance.
Pour cela, modifiez directement les couleurs (en particulier --ske-main-color) dans le fichier skeleton.css.

Variable Utilisation
--ske-main-color Couleur principale du thème.
--ske-light-txt Couleur des textes sur fond foncé.
--ske-dark-bg Couleur des fonds foncés.
--ske-dark-txt Couleur des textes sur fond clair.
--ske-light-bg Couleur des fonds clairs.
--ske-border-color Couleur des bordures.
--ske-alert-bg Couleur de fond des alertes d'erreur.
--ske-alert-txt Couleur de texte des alertes d'erreur.
--ske-warning-bg Couleur de fond des alertes.
--ske-warning-txt Couleur de texte des alertes.
--ske-success-bg Couleur de fond des alertes de succès.
--ske-success-txt Couleur de texte des alertes de succès.

Utilitaires

Skeleton A11y possède plusieurs classes qui servent à aider le design sans avoir à les redéfinir vous-même.

Classe Utilisation
.full-width Faire prendre toute la largeur du conteneur à un élément.
.max-full-width Faire prendre au maximum toute la largeur du conteneur à un élément (utile pour les images).
.margin-top Applique une marge supérieure de 1rem.
.margin-bottom Applique une marge inférieure de 1rem.
.no-padding Supprime les paddings sur une colonne.
.hidden Cache un élément.
.sr-only Cache du texte pour qu'il ne soit lisible que pour les lecteurs d'écrans.
